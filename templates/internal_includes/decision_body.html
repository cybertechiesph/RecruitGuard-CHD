{% load recruitment_ui %} {% comment %} Decision stage — COS path only (2-step wizard for FinalDecisionForm). Step 1 Review the submission packet (read-only context) Step 2 Record your decision (Selected / Not selected; permanent) This template is rendered when application_detail.html sees a FinalDecisionForm in context. That happens for COS cases at HRM Chief Review — the HRM Chief records the COS final decision. Plantilla cases render final_selection_body.html instead at their Appointing Authority Review step: the appointee is picked from the finalized, ranked CAR, not from a single-applicant Selected / Not selected form. Step 1 has no validation — clicking "Next" is the acknowledgement. Step 2's outcome is destructive and permanent, so the existing confirm-modal pattern stays (rewritten in plain English). The backend now blocks recording a decision when the submission packet is missing any required component, so the missing-items banner reads as a hard stop, not a soft caveat. Bootstrap-Icons (.bi-*) classes from the original template aren't loaded by this project, so they were rendering as empty spans. They've been replaced with inline SVGs where they carry meaning, and dropped where they were purely decorative. {% endcomment %} {% if submission_packet %} {# ── Step rail ──────────────────────────────────────────────── #}
